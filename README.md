# FocuxAI Engine™ — HubSpot Adapter v2

**Deterministic. Auditable. Unstoppable.**

## What it does

Takes a Focux Config JSON (generated by FocuxAI Ops wizard) + a HubSpot Private App Token and deploys a complete HubSpot portal configuration:

1. **Property Groups** — "Focux" group for contacts and deals
2. **Contact Properties** — 14+ custom properties with `_fx` suffix
3. **Deal Properties** — 23 custom properties
4. **Sales Pipeline** — with stages and probabilities
5. **User Resolution** — maps asesor emails → HubSpot user IDs
6. **Verification** — reads back portal state to confirm deployment
7. **Workflows (API v4)** — qualification, assignment (round robin), deal creation, value on option, inactivity alerts

## Architecture

```
Browser (Adapter UI)
  ↓ POST /api/hubspot/crm/v3/properties/contacts
Next.js API Route (proxy)
  ↓ POST https://api.hubapi.com/crm/v3/properties/contacts
HubSpot REST API
```

The proxy solves CORS — HubSpot API doesn't accept browser-origin requests.

## Setup

```bash
npm install
npm run dev
```

Open `http://localhost:3000/adapter`

## Deploy to Vercel

```bash
vercel --prod
```

Then add custom domain: `engine.focux.co`

## HubSpot Private App Scopes Required

- `crm.schemas.contacts.write`
- `crm.schemas.deals.write`
- `crm.objects.deals.write`
- `crm.objects.contacts.write`
- `crm.objects.owners.read`
- `automation`

## Stack

- Next.js 14 + React 18
- Vercel (serverless)
- HubSpot CRM REST API v3
- HubSpot Workflows API v4 (beta)
